# Catalogo de Productos Individuales 2026

Objetivo: definir el catalogo inicial de venta para 123tutela en la etapa de cobros individuales con Wompi.

## Principios comerciales

- El analisis inicial del caso y la identificacion del derecho vulnerado son gratis.
- Antes de pagar, el usuario recibe un informe gratuito con analisis, derecho vulnerado y recomendacion.
- En esta etapa no se ofrecen combos.
- En esta etapa no se ofrece `Guia de radicacion` como addon separado.
- En esta etapa no se ofrece `Documento urgente` como addon separado.
- El usuario puede elegir entre:
  - `Solo documento`
  - `Documento + radicacion`
- La promesa comercial es: analizar el caso, generar el documento y radicarlo en menos de 5 minutos cuando el usuario compre la opcion con radicacion y exista canal operativo compatible.
- La plataforma cuenta con base de correos de juzgados y entidades en Colombia para soportar la radicacion donde aplique.
- El panel del cliente debe mostrar historial, documentos, pagos, radicaciones y siguientes pasos sugeridos.

## Reglas de precios

- Todos los precios estan en COP.
- `Documento + radicacion` = `Precio base + 34.000`.
- Los pasos posteriores del caso se ofrecen por evento, no por suscripcion.
- Si un producto posterior aplica, se informa en panel y en correo junto con su valor.

## Oferta comercial base

1. Analisis gratis del caso
   El usuario describe su situacion o adjunta soportes. La plataforma identifica el derecho vulnerado, resume el problema y recomienda la accion juridica mas fuerte.

2. Venta del producto principal
   El usuario elige pagar solo el documento o documento con radicacion.

3. Continuidad del caso
   Despues del radicado o del resultado, el panel y los correos muestran siguientes pasos posibles: seguimiento, impugnacion, desacato u otros tramites relacionados.

## Descripciones detalladas por producto

### Carta formal a entidad

Se usa cuando el usuario necesita pedir, reclamar, advertir o dejar constancia por escrito ante una EPS, banco, empleador, colegio, comercio u otra entidad. Es el punto de entrada mas simple para casos que todavia no requieren una accion constitucional.

### Queja formal

Se usa cuando el usuario necesita denunciar una mala atencion, un incumplimiento o una irregularidad ante una entidad de control o ante la entidad que debe responder. Es util en salud, consumo, servicios publicos y administracion.

### Reclamo administrativo

Se usa para exigir correcciones, respuestas o soluciones frente a un servicio, una factura, una actuacion o una decision administrativa. Es un producto mas fuerte que una simple carta y sirve para dejar trazabilidad del reclamo.

### Derecho de peticion

Se usa cuando el usuario necesita una respuesta oficial, informacion, copias, explicaciones o una actuacion concreta. Es uno de los productos centrales del catalogo y sirve como paso previo frecuente antes de tutela.

### Habeas data

Se usa cuando el problema es informacion personal errada, reportes en centrales de riesgo, datos desactualizados o vulneracion de derechos de correccion o supresion de datos.

### Recurso de reposicion o apelacion

Se usa cuando ya existe una decision previa y el usuario necesita pedir que sea revisada o revocada. Es especialmente util en tramites administrativos, decisiones de entidades y respuestas negativas.

### Queja disciplinaria

Se usa cuando el usuario necesita denunciar faltas o conductas irregulares de funcionarios o servidores. Debe dejar claro que no es representacion judicial sino preparacion del documento y radicacion si se compra la opcion correspondiente.

### Accion de cumplimiento

Se usa cuando una autoridad esta incumpliendo una ley o un acto administrativo obligatorio. Es un producto tecnico y debe explicarse en lenguaje simple para que el usuario entienda cuando aplica.

### Accion de tutela

Se usa cuando hay afectacion o amenaza a un derecho fundamental y se requiere una respuesta urgente. Es uno de los productos mas importantes de la plataforma y debe resaltar la disponibilidad de radicacion usando la base de juzgados y canales operativos.

### Impugnacion de tutela

Se usa cuando ya hubo una decision de tutela y el usuario necesita controvertirla. Debe venderse como paso siguiente natural del caso, no como producto aislado sin contexto.

### Incidente de desacato

Se usa cuando existe fallo favorable de tutela pero la entidad no ha cumplido. Debe mostrarse como continuidad clara del proceso y como producto de alto valor practico.

### Accion popular

Se usa cuando la afectacion es colectiva y compromete intereses de un grupo o de la comunidad. Debe advertirse claramente que no todos los casos individuales encajan en esta accion.

## Reglas de experiencia para producto y pagos

- Antes del pago siempre debe mostrarse:
  - derecho vulnerado detectado
  - recomendacion del producto
  - descripcion clara del producto
  - precio de `Solo documento`
  - precio de `Documento + radicacion`
- Despues del pago aprobado:
  - se genera el documento final
  - si el usuario compro radicacion, se ejecuta y se envia comprobante al correo
  - el panel refleja estado de pago, documento y radicacion
- Despues del radicado:
  - el correo debe informar que puede haber pasos posteriores cobrables por evento
  - el panel debe mostrar el siguiente paso sugerido
